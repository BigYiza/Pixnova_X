# 审核相关配置

> 这里记录的是合规相关配置和重构注意事项，不包含规避审核或绕过封禁的策略。

## 当前审核状态配置

接口：`GET /openapi/query_status_by_ver`

字段：

- `data.is_reviewing`

本地逻辑：

```swift
AppCanHappyShow = !(is_reviewing ?? true)
```

影响：

- 会员页 AB 和关闭按钮策略。
- `IAPManager.resetUserProduct()`。
- 部分促销/展示逻辑。

## 新产品处理建议

- 如果保留该接口，应定义为“审核/合规模式下的展示降级”，例如隐藏实验性促销和不稳定功能。
- 不应将其用于展示与实际审核版本不一致的功能。
- 审核期间可关闭未验证的远程运营弹窗、过激营销文案、外链跳转等。

## 隐私与权限

当前涉及权限：

- 相机。
- 相册读取/保存。
- 推送通知。
- ATT/广告追踪。
- 网络访问。

需要重新确认：

- Info.plist 权限文案。
- App Privacy 填报。
- 第三方 SDK 数据收集说明。
- 用户上传内容处理说明。
- 删除账号/删除内容机制如产品需要。

## 法务文档

当前设置页内置：

- Privacy Policy。
- Terms of Use。
- Subscription Information。

重构必须替换：

- App 名称。
- 公司主体。
- 联系方式。
- 第三方服务列表。
- AI 生成素材处理方。
- 订阅与取消说明。

## 审核高风险点

- 订阅价格和 UI 表达不一致。
- 免费试用/折扣表达不清晰。
- 诱导评分或过早弹评分。
- 权限申请缺少上下文。
- 外链跳转到 TikTok/Facebook/Notion 等未说明用途。
- 用户上传照片后没有清晰说明云端处理。
- 老产品品牌或素材残留。
