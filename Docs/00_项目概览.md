# 项目概览

## 开发重点
这是一个马甲包！！！文档中的信息都基于原项目梳理，严禁使用下面的命名内容，关注重构准则

## 工程定位

当前工程是一个 UIKit 混合的 iOS 内容生成 App。主要能力包括：

- 首页运营内容与模板分发。
- AI Filter、AI Photo、AI Video、Avatar、Baby、图片增强、视频分辨率提升。
- Gemini 流式问答/图文生成。
- 会员订阅、钻石消耗、Apple IAP、权益恢复。
- 个人中心作品历史、下载、分享、删除。
- 奖励中心、邀请奖励、推送通知。
- Firebase、AppsFlyer、SolarEngine、TikTok、Facebook、自研 CodoonAnalytics 等集成。

## 启动链路

1. `PixnovaApp` 注册 `AppDelegate`。
2. `AppDelegate.didFinishLaunching` 调用：
   - Facebook 初始化。
   - `LaunchConfig.share.configWithAppLaunch()`。
   - 注册推送。
   - 注册后台通知。
   - 创建本地数据库表。
3. `ContentView` 先展示 2.5 秒启动页。
4. 首次启动展示引导页 `GuideIntroduceView`。
5. 非首次启动进入 `HomePage`。

## 新工程建议

- 将当前业务拆成 `Network`、`Account`、`Generation`、`Paywall`、`ContentConfig`、`Profile`、`Reward`、`Analytics`、`Compliance` 等模块。
- 后端契约保持兼容，UI、品牌、素材、文案、隐私披露和产品叙事重新设计。
- 用户资产相关模块优先保证稳定：账号、token、IAP、钻石、会员、历史作品。

例如原来：
```

第二层：命名全部替换

例如：

HomeViewController

改：

DiscoverViewController
UserManager

改：

AccountService
NetworkManager

改：

APIClient

这个工作非常适合：

Cursor
Claude Code
Codex

批量完成。

第三层：架构升级

如果原项目是：

MVC

建议直接变：

MVVM

或者：

Clean Architecture Lite

结构：

Feature

  Home
    HomeView
    HomeViewModel
    HomeRepository

  Profile
    ProfileView
    ProfileViewModel

这样代码相似度会大幅下降。

第四层：UI重做

这是苹果最关注的。

如果首页长这样：

Banner

Category

Feed

那新项目不要只是换颜色。

建议：

原
Banner
↓
功能入口
↓
列表

改

推荐内容
↓
卡片瀑布流
↓
专题

布局结构变化比颜色变化更重要。

第五层：网络层重写

AI最擅长干这个。

例如原来：

AFNetworking

或者：

Alamofire

改：

URLSession + async/await

例如：

原代码

NetworkManager.shared.request(...)

改：

await apiClient.send(...)

这里代码重复率会下降很多。
