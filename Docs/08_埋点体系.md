# 埋点框架

## 架构

业务代码只依赖 `AnalyticsTracking`，事件进入 `AnalyticsPipeline` 后在独立串行队列中分发给多个 `AnalyticsDestination`：

```text
业务 / UIKit 自动采集 / IAP / AppMonitor / APIClient
                         |
                 AnalyticsPipeline
                  /      |       \
          Firebase     Console   SolarEngine / 后续引擎
```

- 具体 SDK 故障不会阻塞主线程和业务流程。
- 每个事件自动附加 `event_id`、`session_id`、App/系统/设备/区域信息。
- 事件名会转换为 GA 兼容格式；敏感字段和过长值在总线层统一过滤。
- `setUserID` 与 `flush` 由总线广播给所有引擎。

核心文件：

- `Domain/Services/AnalyticsTracking.swift`：事件模型、统一协议、IAP 状态。
- `Infrastructure/Services/AnalyticsPipeline.swift`：公共属性、清洗、异步扇出。
- `Infrastructure/Services/AnalyticsDestinations.swift`：Firebase 与闭包适配器。
- `Infrastructure/Services/SolarEngineAnalyticsDestination.swift`：热力引擎埋点、归因、账号、IAP、ATT 和 Deep Link 适配。
- `Core/UI/AnalyticsAutoInstrumentation.swift`：全局页面和点击采集。
- `Application/AnalyticsAppMonitor.swift`：运行状态与响应质量。

## 自动覆盖的事件

| 事件名 | 分类 | 内容 |
| --- | --- | --- |
| `ui_click` | interaction | 元素、页面、控件类型/层级路径、绝对坐标、归一化坐标、屏幕尺寸 |
| `page_view` | screen | 页面名、控制器、展示方式、是否模态 |
| `iap_status` | iap | 商品类型、商品/订单/交易 ID、完整购买状态、错误信息 |
| `app_state` | lifecycle | 启动、活跃、非活跃、前台、后台、终止、会话时长 |
| `app_response` | performance | 主线程响应延迟与严重等级 |
| `memory_warning` | performance | 系统内存告警 |
| `thermal_state_changed` | performance | nominal/fair/serious/critical |
| `network_response` | network | path、method、host、状态码、耗时、响应大小和结果 |

自动点击只把位移不超过 14pt、持续不超过 1.2 秒的触摸视为点击，避免把滚动计入热力图。点击不会采集输入内容；元素优先使用 `analyticsElementID` 或 `accessibilityIdentifier`。

IAP 状态包括：

```text
products_loading -> products_loaded/products_load_failed
order_creating -> order_created -> store_presented
-> purchased -> verified -> server_validated -> finished
-> cancelled/pending/failed
restore_started -> restore_finished/restore_failed
```

## 业务埋点

普通业务事件保持简洁：

```swift
analytics.record(
    AnalyticsEvent(
        name: "generation_completed",
        properties: ["template_id": template.id]
    )
)
```

需要稳定的页面或元素名称时显式设置：

```swift
viewController.analyticsScreenName = "membership_paywall"
purchaseButton.analyticsElementID = "membership_purchase"
```

含隐私内容的控件或页面可以关闭自动采集：

```swift
textInput.analyticsTrackingDisabled = true
privatePage.analyticsTrackingDisabled = true
```

## SolarEngine

SolarEngine 已作为独立 Destination 注册，统一接收页面、点击、运行、网络、业务和 IAP 事件；完成的 IAP 还会转换为 SDK 预置 IAP 事件。SDK 自身的自动点击/页面采集保持关闭，避免和统一框架重复上报。

启动顺序为：`preInit`（不采集）→ 用户同意分析与归因 → 注册归因回调 → `start` → 通过 SDK 包装方法请求 ATT。归因结果会记录为 `solar_attribution`，SDK 初始化结果会记录为 `solar_init`。

## 新增 GA、热力图或自研引擎

引入 SDK 后只实现 `AnalyticsDestination`，再在 `DependencyContainer.live()` 的 `destinations` 数组注册，不需要改业务代码：

```swift
final class HeatmapDestination: AnalyticsDestination {
    let identifier = "heatmap"

    func send(_ event: AnalyticsEvent) {
        guard event.name == "ui_click" else { return }
        HeatmapSDK.track(event.properties)
    }
}
```

尚未引入编译依赖时可以先用 `ClosureAnalyticsDestination` 桥接。其他热力图引擎可直接消费 `x_ratio`、`y_ratio`、`screen_name` 和 `element_name`。

## 约束

- 禁止上报密码、token、receipt、授权 Header；总线还会按 key 二次过滤。
- 不把邮箱、手机号、用户输入或完整 URL 放进属性。
- 新事件使用小写下划线命名，属性值长度不超过 500 字符。
- IDFA 必须在 ATT 授权和隐私披露完成后，通过独立属性提供器接入。
- Debug 控制台用于联调；生产数据以已注册 Destination 为准。
