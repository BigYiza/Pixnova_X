# 埋点体系

## 双通道

`AnalyticTool.logEvent` 同时上报：

- Firebase Analytics
- 自研 CodoonAnalytics

## 通用事件名

| 事件 | 含义 |
| --- | --- |
| `app_install` | 安装/首次引导 |
| `app_start` | 启动 |
| `app_end` | 上次使用时长 |
| `exposure` | 页面/弹窗曝光 |
| `click` | 点击 |
| `operation` | 业务操作 |

## 通用参数

| 参数 | 含义 |
| --- | --- |
| `event_type` | 事件类型 |
| `element_name` | 元素名 |
| `current_page_name` | 当前页 |
| `from_element_name` | 来源元素 |
| `redundancy_info` | 扩展信息 |
| `operation_type` | 操作类型 |
| `operation_result` | 操作结果 |
| `source` | 来源 |
| `service_time` | 服务耗时 |

## 自研上报机制

CodoonAnalytics：

- 本地 SQLite：`analytics.sqlite`
- 表：`CodoonAnalyticEventsTable`
- 每批最多 10 条。
- 每 60 秒上报一次。
- 未上报事件达到 10 条时立即上报。
- 失败最多重试 3 次，每次延迟 5 秒。
- 已上传事件 7 天后清理。

## 自研上报接口

| 接口 | 说明 |
| --- | --- |
| `/analyze/openapi/pixnova/report_preset_properties` | 上报预置属性 |
| `/analyze/openapi/pixnova/report_event_data` | 上报事件 |

签名 header：

- `X-Client-Id`
- `Timestamp`
- `Nonce`
- `X-Api-Signature`

签名方式：按 `client_id/path/timestamp/nonce/body` 排序拼接后 HMAC-SHA256。

## 预置属性

包括：

- os、os_version
- app_name、app_version
- manufacturer、device_type、device_model
- device_id
- region、language、timezone
- channel
- idfa
- user_id

## 新工程建议

- 做一个 Analytics Adapter，业务只依赖统一协议。
- 对新产品重新命名页面和事件，保留关键漏斗语义。
- 不要把老产品品牌名写死到新埋点。
- IDFA 上报必须受 ATT 和隐私披露约束。
