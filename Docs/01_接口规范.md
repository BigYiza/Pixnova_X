## Header

普通接口由 `RBXServer.headers` 统一生成：

| Header | 说明 |
| --- | --- |
| `Authorization` | `Bearer {accessToken}` |
| `tokenId` | accessToken |
| `uid` | userId |
| `timezone` | 当前时区分钟偏移 |
| `User-Agent` | `pixnova;版本;iOS;系统版本;机型;Apple Store;时区;语言` |
| `device-id` | Keychain/UserDefaults 持久化 UUID |
| `Content-Type` | JSON 或 multipart |
| `Clt-Start` | 仅生成接口使用，UTC 时间 |

## 网络请求层实现要求

新工程只要后端不变，网络层至少要完整复刻以下逻辑，否则会出现登录后仍 401、支付接口解析失败、生成任务失败或 Gemini 不能流式返回的问题。

### 1. BaseURL 与 path

请求分三类：

| 类型 | BaseURL | path |
| --- | --- | --- |
| 普通业务接口 | `SERVER_HOST` | API path，如 `/api/query_cards` |
| 支付接口 | 支付 host | 支付 path，如 `/api/v1/order/create` |
| 下载接口 | 完整下载 URL | 空字符串 |

实现规则：

```swift
func baseURL(for api: API, requestType: RequestType) -> URL {
    if case .download = requestType {
        return URL(string: api.rawValue)!
    }
    if let specialHost = api.specialHost {
        return URL(string: specialHost)!
    }
    return URL(string: SERVER_HOST)!
}

func path(for api: API, requestType: RequestType) -> String {
    if case .download = requestType {
        return ""
    }
    return api.rawValue
}
```

### 2. Method 与参数编码

当前约定：

| Method | Encoding | 说明 |
| --- | --- | --- |
| POST | JSON body | `JSONEncoding.default` |
| GET | URL query | `URLEncoding.default` |
| Upload multipart | multipart body + urlParameters | 老上传封装，当前主流程更多使用 OSS/Firebase |
| OSS PUT | binary body | 不走 Moya 业务封装 |

实现规则：

```swift
let encoding: ParameterEncoding = method == .post
    ? JSONEncoding.default
    : URLEncoding.default
```

没有参数时使用 plain request；有参数时按 method 编码。

### 3. Header 生成规则

普通业务接口、支付接口都要带相同的基础 header。登录前没有账号时，只是不带鉴权三件套，其他 header 仍要带。

```swift
func makeHeaders(api: API, requestType: RequestType, baseURL: URL) -> [String: String] {
    var headers: [String: String] = [:]

    if AccountManager.default.currentAccount.isValid() {
        let token = AccountManager.currentUserToken()
        headers["Authorization"] = "Bearer \(token)"
        headers["tokenId"] = token
        headers["uid"] = AccountManager.currentUserId()
    }

    let timezoneMinutes = TimeZone.autoupdatingCurrent.secondsFromGMT() / 60
    headers["timezone"] = String(timezoneMinutes)

    let version = Bundle.main.infoDictionary?["CFBundleShortVersionString"] as? String ?? ""
    let systemVersion = UIDevice.current.systemVersion
    let languageCode = Locale.current.language.languageCode?.identifier ?? "en"
    let deviceCode = DeviceUtil.deviceCode

    headers["User-Agent"] = "pixnova;\(version);iOS;\(systemVersion);\(deviceCode);Apple Store;\(timezoneMinutes);\(languageCode)"
    headers["device-id"] = UUIDManager.getUUID()

    switch requestType {
    case .request, .download:
        headers["Content-Type"] = "application/json"
    case .upload:
        headers["Content-Type"] = "multipart/form-data"
    }

    if api == .generaImage {
        headers["Clt-Start"] = utcString(Date())
    }

    return headers
}

func utcString(_ date: Date) -> String {
    let formatter = DateFormatter()
    formatter.dateFormat = "yyyy-MM-dd'T'HH:mm:ss'Z'"
    formatter.timeZone = TimeZone(secondsFromGMT: 0)
    return formatter.string(from: date)
}
```

重点：

- `Authorization` 必须是 `Bearer ` + token，中间有空格。
- `tokenId` 和 `uid` 也要同时传，后端当前会用到。
- `timezone` 是分钟，不是小时。
- `User-Agent` 字段里的产品名当前固定为 `pixnova`；如果新产品后端仍按该 UA 识别，先不要擅自改。若要改名，需要和后端确认兼容。
- `/api/image_generation` 必须加 `Clt-Start`，否则可能影响服务端耗时统计或风控。

### 4. 普通接口响应解析

普通业务接口必须按以下规则解析：

```swift
func handleBusinessResponse(_ json: [String: Any]) -> Result<[String: Any], APIProviderError> {
    guard let code = json["code"] as? Int else {
        return .failure(.mappingError)
    }

    if code == 0 {
        return .success(json)
    }

    if code == -100 {
        AccountManager.default.refreshToken()
        return .failure(.serverError("Token Failed", -100))
    }

    let message =
        json["description"] as? String ??
        json["desc"] as? String ??
        json["msg"] as? String ??
        String(localized: "request fail")

    return .failure(.serverError(message, code))
}
```

HTTP 层：

- 非 2xx：当作 `mappingError`。
- JSON 不能转字典：当作 `mappingError`。
- Moya/URLSession 请求失败：当作 `requestError`。

### 5. 支付接口响应解析例外

支付 host 包含：

- `https://pay-vmddzvrudq-df.a.run.app`
- `https://pay-mhsaciltta-wl.a.run.app`

支付接口不能套普通 `code == 0` 校验。当前逻辑是：

```swift
if baseURL.absoluteString.contains(PaymentBaseUrl_Debug)
    || baseURL.absoluteString.contains(PaymentBaseUrl_Release) {
    return .success(jsonDictionary)
}
```

业务层再自行读取：

- 创建订单：`data.order_id`, `data.order_uuid`
- 查询订单：`data.state`
- 恢复购买：顶层 `state == 0`

### 6. Object / Array 映射

普通业务成功后有三种读取方式：

| 方法 | 用途 | 规则 |
| --- | --- | --- |
| `api_request` | 业务自己解析字典 | 返回整个 JSON |
| `api_requestObject` | 映射对象 | 默认取 `json["data"]`，没有则用整个 JSON |
| `api_requestArray` | 映射数组 | 默认取 `json["data"] as [[String: Any]]` |

新工程如果不用 ObjectMapper，也要保留相同的取 key 规则。

### 7. OSS 上传请求

OSS 上传分两步。

第一步：向业务后端取签名。

```json
{
  "content_type": "image/jpeg",
  "filename": "UUID.jpeg"
}
```

响应：

```json
{
  "code": 0,
  "data": {
    "sign_url": "https://...",
    "file_url": "https://..."
  }
}
```

第二步：对 `sign_url` 直接发 PUT。

```swift
var request = URLRequest(url: URL(string: signURL)!)
request.httpMethod = "PUT"
request.setValue(contentType, forHTTPHeaderField: "Content-Type")
request.httpBody = data
```

HTTP 2xx 后，把 `file_url` 作为素材 URL 传给 `/api/image_generation` 的 `urls`。

### 8. Gemini SSE 请求

Gemini 不能走 Moya 普通请求，要用 `URLSessionDataDelegate`。

URL：

```text
POST {SERVER_HOST}/api/chat_generation_content
```

Header：

| Header | 值 |
| --- | --- |
| `Authorization` | `Bearer {token}`，账号有效时 |
| `tokenId` | token，账号有效时 |
| `uid` | userId，账号有效时 |
| `timezone` | 当前时区分钟 |
| `User-Agent` | `pixnova;版本;iOS;系统版本;机型;Apple Store;时区` |
| `device-id` | UUID |
| `Content-Type` | `application/json` |
| `Accept` | `text/event-stream` |

Body：

```json
{
  "content": "用户输入",
  "images": ["可选图片 URL"],
  "session_id": "可选会话 id"
}
```

响应处理：

- 从 response header 读取 `session_id` 并保存。
- data chunk 中提取顶层 JSON 对象。
- `type == "text"`：追加 `text`。
- `type == "images"`：读取 `images[].url`。
- `code == -101`：打开会员页。

### 9. 请求层最小实现清单

新工程网络层至少要实现：

- `API` path 枚举，支持 `specialHost`。
- `SERVER_HOST` / 支付 host 环境配置。
- GET query / POST JSON 编码。
- 鉴权 header。
- timezone / User-Agent / device-id。
- `/api/image_generation` 的 `Clt-Start`。
- 普通业务 `code` 解析和 `-100` 刷 token。
- 支付 host 解析例外。
- Object/Array 的默认 `data` 映射。
- OSS PUT。
- Gemini SSE。

## 特殊接口

- OSS `sign_url` 不是业务接口，直接 `PUT` 二进制数据。
- Firebase Storage 上传不走后端接口，上传成功后使用 download URL。
- Gemini 是流式响应，不能用 Moya 的普通 JSON 解析。
- 支付域名响应不按普通 `code == 0` 规则拦截。
