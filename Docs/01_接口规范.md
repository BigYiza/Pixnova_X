

## Header

普通接口由 `RBXServer.headers` 统一生成：
普通业务接口由 `RBXServer.headers` 统一生成：

| Header | 说明 |
| --- | --- |
| `Content-Type` | JSON 或 multipart |
| `Clt-Start` | 仅生成接口使用，UTC 时间 |

支付接口也走 `RBXServer.headers`，但请求构建时必须先切到支付 `baseURL`。因此 HTTP 请求的 URL、TLS SNI 和自动生成的 `Host` header 都是支付域名，不是 App 业务域名。

| 场景 | BaseURL | 自动 `Host` header | 显式业务 header |
| --- | --- | --- | --- |
| App 业务接口 | `https://api.pixnova.app` | `api.pixnova.app` | `Authorization`、`tokenId`、`uid`、`timezone`、`User-Agent`、`device-id`、`Content-Type`，生成接口额外 `Clt-Start` |
| 支付 Release 接口 | `https://pay-mhsaciltta-wl.a.run.app` | `pay-mhsaciltta-wl.a.run.app` | 同一套显式业务 header，但必须基于支付 `baseURL` 发起请求 |
| 支付 Debug 接口 | `https://pay-vmddzvrudq-df.a.run.app` | `pay-vmddzvrudq-df.a.run.app` | 同一套显式业务 header，但必须基于支付 `baseURL` 发起请求 |

注意：

- 不要手动写死 HTTP `Host` header；Moya/URLSession 会根据请求 URL 自动生成。
- 支付接口不能拼成 `https://api.pixnova.app/api/v1/...`，也不能复用 App host 的 `Host: api.pixnova.app`。
- 支付环境 Release 正式地址是 `https://pay-mhsaciltta-wl.a.run.app`，不是 `https://pay-mhsaciltta-wl.a.run.p`。

## 网络请求层实现要求

新工程只要后端不变，网络层至少要完整复刻以下逻辑，否则会出现登录后仍 401、支付接口解析失败、生成任务失败或 Gemini 不能流式返回的问题。

### 3. Header 生成规则

普通业务接口、支付接口都要带相同的基础 header。登录前没有账号时，只是不带鉴权三件套，其他 header 仍要带。
普通业务接口、支付接口都要带相同的显式业务 header。登录前没有账号时，只是不带鉴权三件套，其他 header 仍要带。

支付接口的差异不在 `Authorization/timezone/User-Agent/device-id/Content-Type` 这些显式字段，而在请求 URL 所属的 `baseURL`：支付 path 必须通过 `API.specialHost` 使用支付 host。HTTP 层的 `Host` header 会随 URL 变成 `pay-mhsaciltta-wl.a.run.app` 或 `pay-vmddzvrudq-df.a.run.app`，不能沿用 `api.pixnova.app`。

```swift
func makeHeaders(api: API, requestType: RequestType, baseURL: URL) -> [String: String] {
    let languageCode = Locale.current.language.languageCode?.identifier ?? "en"
    let deviceCode = DeviceUtil.deviceCode

    if baseURL.absoluteString.contains(PaymentBaseUrl_Debug)
        || baseURL.absoluteString.contains(PaymentBaseUrl_Release) {
        // 支付请求必须已经通过 specialHost 切到支付 baseURL。
        // 不要手动设置 Host；URLSession 会使用支付域名生成 Host/SNI。
    }

    headers["User-Agent"] = "pixnova;\(version);iOS;\(systemVersion);\(deviceCode);Apple Store;\(timezoneMinutes);\(languageCode)"
    headers["device-id"] = UUIDManager.getUUID()

- `tokenId` 和 `uid` 也要同时传，后端当前会用到。
- `timezone` 是分钟，不是小时。
- `User-Agent` 字段里的产品名当前固定为 `pixnova`；如果新产品后端仍按该 UA 识别，先不要擅自改。若要改名，需要和后端确认兼容。
- 支付接口的 URL host、HTTP `Host` header 和 TLS SNI 必须是支付域名；不要用 App host 代理或覆盖。
- `/api/image_generation` 必须加 `Clt-Start`，否则可能影响服务端耗时统计或风控。

### 4. 普通接口响应解析

- `API` path 枚举，支持 `specialHost`。
- `SERVER_HOST` / 支付 host 环境配置。
- 支付请求使用支付 `baseURL`，让 HTTP `Host` 自动变成支付域名，不能使用 `api.pixnova.app` 的 host。
- GET query / POST JSON 编码。
- 鉴权 header。
- timezone / User-Agent / device-id。
