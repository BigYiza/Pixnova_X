# 支付体系

## 支付模块

| 文件 | 职责 |
| --- | --- |
| `IAPManager.swift` | StoreKit 商品、购买、恢复、交易监听 |
| `IAPService.swift` | 后端订单和验单 |
| `IAPModel.swift` | 商品和交易模型 |
| `MemberShipViewModel.swift` | 会员购买业务 |
| `DiamondPurchaseModel.swift` | 钻石购买业务 |

## 购买流程

```text
加载 SKProduct
  -> 后端创建订单
  -> StoreKit 发起购买
  -> 读取 receipt
  -> 后端 success_notify 验证
  -> finishTransaction
  -> 查询订单状态
  -> 刷新会员/钻石状态
```

## 订单接口

创建订单参数：

```json
{
  "name": "",
  "description": "",
  "product_id": "",
  "purchase_price": ""
}
```

成功响应：

- `data.order_id`
- `data.order_uuid`

`order_uuid` 会作为 `applicationUsername` 传给 Apple。

## 验单接口

```json
{
  "order_id": "",
  "transaction_id": "",
  "receipt_data": ""
}
```

## 订单状态

- `/api/v1/order/status`
- `state == 1` 表示完成。
- 最多轮询 5 次。

## 恢复购买

恢复流程：

1. StoreKit restore。
2. 取 original transaction id。
3. 后端 `/api/v1/pay/apple/restore`。
4. `state == 0` 成功。
5. 必要时调用 `/api/restore_user_id`。

## 风险点

- 支付域名响应不按普通 `code` 校验。
- 商品 ID 与 App Store Connect 必须一致。
- 新产品如果更换 bundle id，需要重新配置商品或确认跨 App 后端策略。
- 恢复购买涉及用户资产，必须回归测试。
