# 会员体系

## 会员状态接口

接口：`GET /api/query_vip_status`

关键字段：

| 字段 | 说明 |
| --- | --- |
| `is_vip` | 大于 0 表示 VIP |
| `expires_time` | VIP 到期 |
| `video_times` | 视频次数 |
| `give_ai_videos_times` | 赠送视频次数状态 |
| `free_vip_times` | 免费权益次数 |
| `diamonds` | 钻石余额 |

刷新后会更新本地 Account，并发送 `MembershipStateDidChanged`。

## 会员页 AB

依赖：

- `/openapi/query_status_by_ver` 的 `is_reviewing`
- `/api/query_user_group_by_pos` 的 `MemberShipCloseBtn`
- `/api/query_user_group_by_pos` 的 `MemberShip_Paywall`

如果 `AppCanHappyShow == false`，会员页使用 Normal 方案。

## 关闭按钮策略

| 分组 | 说明 |
| --- | --- |
| `MemberShipCloseBtn_Normal` | 普通关闭 |
| `MemberShipCloseBtn_Close_3s` | 3 秒后展示关闭 |
| `MemberShipCloseBtn_Close_None` | 不展示关闭 |

## 商品分发

会员商品 ID 由用户分组决定：

- Normal：普通周、普通年、3 天试用年。
- Close 3s：对应 closebtn.3sshow 商品。
- None：对应 closebtn.none 商品。

此外还加载：

- `Pixnova.vip.weekly.449.nofreetrail`
- `Pixnova.vip.yearly.online.nofreetrail.newuserdiscount`

## 会员权益影响

当前业务中 VIP 影响：

- VIP 模板是否可用。
- Cutout 是否可用。
- 生成结果是否加水印。
- 保存图片是否加水印。
- 是否可以跳过等待/后台生成。
- 周钻石奖励弹窗。

## 免费权益

`freeVipTimes` 用于 Filter 模块的免费试用弹窗：

- 大于 1：弹样式 1，可试用或去会员。
- 等于 1：弹样式 2。
- 等于 0：直接会员页。

## 新工程注意

- 会员页文案、权益说明、试用说明必须与 App Store Connect 商品一致。
- 会员权益判断应集中，不建议散落在页面。
- 当前代码里“审核状态”影响会员展示，新工程应按合规逻辑重新定义，不应作为规避审核手段。
