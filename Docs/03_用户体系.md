# 用户体系

## 账号模型

`Account` 是本地用户核心模型，保存到 UserDefaults：

- key：`currentUserAccount`
- 格式：ObjectMapper JSON 字符串

主要字段：

| 字段 | 说明 |
| --- | --- |
| `accessToken` | 访问 token |
| `tokenExpireTime` | token 过期时间戳 |
| `userId` | 用户 id |
| `isVip` | 是否会员 |
| `vipExpirationTime` | 会员到期 |
| `videoTimes` | 视频次数 |
| `freeVipTimes` | 免费权益次数 |
| `diamonds` | 钻石 |
| `invitationInfo` | 邀请数据 |
| `vipRewardInfo` | VIP 奖励 |
| `userGroupMap` | AB 分组 |
| `user` | 注册信息 |

## 登录

- 启动时本地账号有效则复用。
- 本地账号无效则调用 `/openapi/login`。
- 成功后保存 `user_id` 和 `token_info.token`。

注意：当前 `isValid()` 是 `userId` 或 `accessToken` 任一存在即有效。新工程建议改成更严格的状态，但要考虑老用户迁移。

## Token 刷新

触发时机：

- App 进入前台。
- token 已过期。
- token 剩余不足 30 分钟。
- 接口返回 `-100`。

刷新失败会重新登录。

## 用户信息

`/api/query_user_info` 返回三块：

- `invite_info`
- `vip_reward`
- `user`

用途：

- 邀请奖励弹窗。
- VIP 周奖励弹窗。
- 邀请码是否可兑换。
- 注册 3 天内邀请码填写规则。

## 用户分组

接口：`/api/query_user_group_by_pos`

当前请求：

```json
{
  "postion_list": ["MemberShipCloseBtn", "MemberShip_Paywall"]
}
```

返回数据直接作为 `userGroupMap`。会员页根据分组切换商品和展示。

## 设备 ID

`UUIDManager.getUUID()` 用于：

- `device-id` header。
- AppsFlyer customerUserID。
- 自研埋点设备标识。

实现上优先 Keychain，失败时 fallback UserDefaults。新包如果更换 bundle/keychain access group，设备 ID 可能重置，这是正常的新产品行为，但后端识别会受影响。
